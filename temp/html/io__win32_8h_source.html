88 // Expand wildcards in a path pattern, feed the result to a consumer function.
89 //
90 // `path` must be a valid, Windows-style path. It may be absolute, or relative
91 // to the current working directory, and it may contain wildcards ("*" and "?")
92 // in the last path segment. This function passes all matching file names to
93 // `consume`. The resulting paths may not be absolute nor normalized.
94 //
95 // The function returns a value from `ExpandWildcardsResult`.
